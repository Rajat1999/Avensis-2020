Code Shuffle

Description

Logical thought process and astute decision-making. It is a one day event which will test your ability to design, understand and debug the code. Participation can be in the form of individuals or a team of two. It will consist of two rounds.
Round 1-
Participants have to take the program to its working state. Programming language will be one of c++ or java .
Round 2-
Participants have to code in an unfamiliar programming language whose documentation will be provided.
The Event will be organised in Maharaja Surajmal Institute of Technology on 8th January ,2020.
Participants must reach the venue by 10 am sharp. Result will be declared on the spot.

Rules

  • Contestants cannot bring any notes or textbooks to the contest room. You have to bring your own paper and pen.
  • Each team will be allowed to use only a single computer and must write and test their programs on that machine, wholely within the contest room.
  • Decision made by the Event heads will be final.
  • Contestants in the same team can discuss the problems they are assigned. However, they should do so in a quiet way and without disturbing contestants in other teams.
  • Each problem will have a specified point value. The more difficult the problem, the more points a correct solution will receive.
  • Ties will be broken by the total time for each user in ascending order of time.

Hack-Xpress(Hackathon)

Description

The sole purpose of this event is to create a usable software based on theme provided.This is a two day event, day one work will end on 5:00pm and resume at 9:00am on 9th January 2020. Participants will report on 8th January at 9:00 am sharp at Maharaja Surajmal Institute of Technology. Team size can varies from 2 to 4 members.Team must contain only college students across Delhi,NCR.
Themes for the event are:
1. Modernizing Education
2. Social Issues
3. Virtual Security
4. Smart Communication
5. Healthcare
6. Urban mobility problems

Rules

  • All code must be written during the event. You cannot use existing code that you have written and finish it during the event.
  • Participants cannot use any external APIs as a part of their core functionality. (Using Google Maps API for an app that tracks a delivery is fine, but using Google Maps API to build a navigation app is not allowed)
  • You can apply as a team of 2-4 members. Teams can consist of up to a maximum of 4 members.
  • The project must be developed in public repositories on GitHub and should have a valid Open Source License.
  • All participants must bring their own laptops and cords, no computers will be provided.
  • The organizers reserve the right to make changes to the event program.
  • The prize pool will be split among the winners.
